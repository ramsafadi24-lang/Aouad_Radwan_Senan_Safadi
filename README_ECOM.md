## Project Overview

This repository contains the full data processing pipeline for the MSBA 305 course project.  
The pipeline ingests, stores, cleans, processes, queries, visualises, and governs three heterogeneous e-commerce data sources, producing actionable business insights from a UK online retailer's transaction history.

**Domain:** E-Commerce Retail Analytics  
**Pipeline Stages:** Acquisition → Storage → Cleaning → Processing → Querying → Visualisation → Governance

## Environment Setup

### Prerequisites

* Python 3.9 or higher (3.11 recommended)
* Or Google Colab (recommended — no local install required)

### Option A: Google Colab (Recommended)

1. Upload `MSBA305\\\_Full\\\_Pipeline.ipynb` to Google Colab.
2. Upload `Online\\\_Retail.xlsx` and `com-amazon.top5000.cmty.txt` to `/content/` in the Colab file browser (left sidebar).
3. Mount Google Drive when prompted (cell 5) for exporting cleaned CSVs.
4. Run all cells in order (`Runtime → Run all`).

> \\\*\\\*Note:\\\*\\\* MongoDB Atlas and Neo4j AuraDB credentials in the notebook are for demonstration. Replace with your own credentials for production use — never commit real credentials to a public repository.

### Option B: Local Python

```bash

# 2. Create and activate a virtual environment
python -m venv .venv

#     COPURCHASE\\\_PATH  = "data/raw/com-amazon.top5000.cmty.txt"

# 6. Launch Jupyter
jupyter notebook MSBA305\\\_Full\\\_Pipeline.ipynb
```

## Technology Decision Summary

|Component|Chosen|Rejected (with reason)|
|-|-|-|
|Ingestion mode|Batch Python scripts|Kafka streaming (no real-time need), Airflow (server overhead unjustified for single-node)|
|Relational storage|SQLite|MongoDB (rigid schema, document model adds no value at 541K rows), ClickHouse (columnar analytics overkill)|
|Document storage|MongoDB Atlas|Firestore (GCP vendor lock-in), DynamoDB (complex pricing, AWS-specific)|
|Graph storage|Neo4j AuraDB|NetworkX (in-memory only, not persistently queryable), Amazon Neptune (expensive)|
|In-memory processing|pandas|Polars (narrower team familiarity), Dask (distributed overhead not warranted < 1M rows)|
|Visualisation|Matplotlib + Plotly|Tableau (requires licence), Power BI (Windows-centric, incompatible with Colab)|

## Known Limitations

* **SQLite concurrency:** Does not support concurrent writes. Designed for a single-analyst environment.
* **Hard-coded credentials:** MongoDB and Neo4j connection strings are embedded for demonstration. Replace with `os.environ` references before deploying to any shared or public environment.
* **Colab path dependency:** File paths assume `/content/` (Google Colab). Update `XLSX\\\_PATH` and `COPURCHASE\\\_PATH` constants for local execution.
* **Synthetic product data:** DummyJSON catalog is fictitious. Product catalog analytics are illustrative and should not be used for real market decisions.
* **Temporal scope:** Transaction data covers Dec 2010 – Dec 2011 only. All insights must be interpreted within this period.
